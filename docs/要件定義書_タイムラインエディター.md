> プロジェクト: medianine
> 作成日: 2026-03-25
> ステータス: ドラフト（レビュー待ち）

---

## 1. プロジェクト概要

| 項目 | 内容 |
|------|------|
| プラグイン名 | medianine-editor（仮） |
| 目的 | Remotionプロジェクト全体で使える、CapCutライクなタイムラインベースの動画編集UI |
| 対象ユーザー | AI-BOWチーム、生徒さん・会員さん |
| 配置場所 | `src/editor/` |
| 技術スタック | React 19 + TypeScript + Tailwind CSS 4 + Remotion 4.0.438 |

### 1.1 解決する課題

現状、動画の構成変更（クリップ並び替え、テロップタイミング調整、カットポイント変更）はすべてコード上で手動編集が必要。CapCutのような直感的なタイムラインUIを提供することで、コードを触らずに動画編集ができるようにする。

### 1.2 対象テンプレート

既存の全テンプレートで共通利用できること：

- **NewsTemplate**（MyComp） — ニュース風動画、字幕・カットポイント付き
- **PokerShortsTemplate** — Shortsテンプレート、テロップ・タイミング制御
- **PhoneTemplate** — シーンベースのモーショングラフィックス

## 2. 機能要件

### 2.1 タイムライン（コア機能）

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| TL-01 | マルチトラック表示 | 必須 | 動画・音声・テロップ・エフェクトを別トラックで表示 |
| TL-02 | プレイヘッド | 必須 | 再生位置を示すヘッド。ドラッグで任意位置にシーク |
| TL-03 | ズーム・スクロール | 必須 | タイムラインの拡大縮小、横スクロール |
| TL-04 | フレーム数/秒数 切替表示 | 必須 | ルーラーにフレーム数 or 秒数を表示 |

### 2.2 クリップ操作

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| CL-01 | クリップのドラッグ&ドロップ | 必須 | トラック上でクリップを移動・並び替え |
| CL-02 | クリップのトリム | 必須 | 端をドラッグしてイン/アウトポイントを変更 |
| CL-03 | クリップの分割（スプリット） | 必須 | プレイヘッド位置でクリップを2つに分割 |
| CL-04 | クリップの削除 | 必須 | 選択したクリップを削除 |
| CL-05 | クリップの複製 | 推奨 | 選択したクリップをコピー |
| CL-06 | トラック間移動 | 推奨 | クリップを別のトラックにドラッグ移動 |

### 2.3 テロップ/字幕編集

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| TX-01 | テロップの配置タイミング調整 | 必須 | タイムライン上でテロップの開始/終了フレームをドラッグ変更 |
| TX-02 | テロップテキスト編集 | 必須 | ダブルクリックでテキスト内容を編集 |
| TX-03 | テロップスタイル変更 | 推奨 | フォントサイズ、色、位置のプロパティパネル |
| TX-04 | テロップの追加/削除 | 必須 | 新規テロップ追加と既存削除 |

### 2.4 音声操作

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| AU-01 | 音声波形表示 | 推奨 | 音声トラックに波形をビジュアル表示 |
| AU-02 | ボリューム調整 | 必須 | クリップ単位でのボリューム変更 |
| AU-03 | ミュート/ソロ | 推奨 | トラック単位でのミュート・ソロ切替 |

### 2.5 カットポイント編集

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| CT-01 | カット範囲の可視化 | 必須 | 既存のcutRangesをタイムライン上にハイライト表示 |
| CT-02 | カット範囲の調整 | 必須 | カット範囲の開始/終了をドラッグで変更 |
| CT-03 | カット範囲の追加/削除 | 必須 | 新規カット追加・既存削除 |
| CT-04 | 無音検出結果の反映 | 推奨 | scripts/detect_silence.pyの結果をインポート |

### 2.6 プレビュー連携

| ID | 機能 | 優先度 | 説明 |
|----|------|--------|------|
| PV-01 | Remotion Player連携 | 必須 | タイムラインとRemotionプレビューの同期再生 |
| PV-02 | 再生/停止コントロール | 必須 | 再生・一時停止・先頭に戻る |
| PV-03 | リアルタイムプレビュー更新 | 必須 | タイムライン変更を即座にプレビューに反映 |

## 3. 非機能要件

| カテゴリ | 要件 |
|----------|------|
| パフォーマンス | 60fpsのタイムラインでスムーズにスクロール・操作可能 |
| レスポンシブ | 最低1280px幅のデスクトップブラウザで動作 |
| 拡張性 | 新しいテンプレート追加時に最小限の変更で対応 |
| データ永続化 | 編集状態をJSONでエクスポート/インポート可能 |
| 依存関係 | Remotion + React + TailwindCSS以外の追加依存は最小限 |

## 4. データ設計

### 4.1 タイムラインステート（JSON構造）

```typescript
interface TimelineState {
  fps: number;
  durationInFrames: number;
  tracks: Track[];
}

interface Track {
  id: string;
  type: "video" | "audio" | "text" | "effect";
  label: string;
  muted: boolean;
  locked: boolean;
  visible: boolean;
  items: TrackItem[];
}

interface TrackItem {
  id: string;
  startFrame: number;
  endFrame: number;
  data: VideoItemData | AudioItemData | TextItemData | EffectItemData;
}

interface VideoItemData {
  type: "video";
  src: string;
  trimStart: number;   // ソース動画内の開始フレーム
  trimEnd: number;      // ソース動画内の終了フレーム
}

interface AudioItemData {
  type: "audio";
  src: string;
  volume: number;       // 0.0 ~ 1.0
  trimStart: number;
  trimEnd: number;
}

interface TextItemData {
  type: "text";
  text: string;
  style: {
    fontSize: number;
    color: string;
    fontFamily: string;
    position: { x: number; y: number };
  };
}

interface EffectItemData {
  type: "effect";
  effectId: string;     // カット、トランジション等
  params: Record<string, unknown>;
}
```

### 4.2 既存テンプレートとの接続

各テンプレートがタイムラインエディターと連携するために、テンプレートアダプターを定義：

```typescript
interface TemplateAdapter {
  // テンプレートのpropsからTimelineStateに変換
  toTimeline(props: Record<string, unknown>): TimelineState;
  // TimelineStateからテンプレートのpropsに変換
  fromTimeline(state: TimelineState): Record<string, unknown>;
}
```

| テンプレート | 対応するアダプター |
|---|---|
| NewsTemplate (MyComp) | cutRanges, subtitles ⇔ Track items |
| PokerShortsTemplate | telops, timings ⇔ Track items |
| PhoneTemplate | SCENES配列 ⇔ Track items |

## 6. キーボードショートカット

| ショートカット | 操作 |
|---------------|------|
| Space | 再生/一時停止 |
| S | プレイヘッド位置で分割 |
| Delete / Backspace | 選択クリップを削除 |
| Ctrl+C / Ctrl+V | コピー/ペースト |
| Ctrl+Z / Ctrl+Y | Undo / Redo |
| ← / → | 1フレーム移動 |
| Shift+← / Shift+→ | 10フレーム移動 |
| +/- | タイムラインズーム |
| M | 選択トラックをミュート |
| L | 選択トラックをロック |

## 7. 開発フェーズ

### Phase 1: 基盤（MVP）
- タイムラインコンポーネント（マルチトラック、プレイヘッド、ルーラー）
- クリップのドラッグ&ドロップ、トリム、分割、削除

- テロップ編集（タイミング調整、テキスト編集、スタイル変更）
- 音声ボリューム調整、ミュート/ソロ
- プロパティパネル
- PokerShortsTemplateアダプター

### Phase 3: 拡張機能
- 音声波形表示
- 無音検出結果のインポート（scripts/detect_silence.py連携）
- PhoneTemplateアダプター
- Undo/Redo
- キーボードショートカット

### Phase 4: 仕上げ
- パフォーマンス最適化
- エラーハンドリング
- ドキュメント整備
- 新テンプレート追加ガイド

## 8. 技術的な制約と判断

| 項目 | 判断 |
|------|------|
| 外部ライブラリ | タイムラインUIは自前実装（Remotion Player以外の依存を最小限に） |
| ドラッグ&ドロップ | HTML5 Drag & Drop API + ポインターイベントで実装 |
| ステート管理 | React useReducer + Context（Redux等は不使用） |
| スタイリング | Tailwind CSS 4（プロジェクト既存のものを利用） |
| ファイル操作 | ブラウザ上のみ、サーバー側不要（JSONベースの永続化） |
